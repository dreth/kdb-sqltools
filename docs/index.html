VS Code extension - current package 0.2.8

Run q from SQLTools and inspect kdb+ results without flooding the DOM.

kdb-sqltools connects VS Code SQLTools to a kdb+/q process over q IPC, executes q and qSQL directly, and ships a dedicated kdb results panel for large tables, focused editor workflows, copy, and export.

Quick start

Install, start q, connect

Requirements

  • VS Code with the mtxr.sqltools extension installed (SQLTools website, GitHub).
  • A kdb+/q process listening on a TCP port.
  • This extension installed from a packaged VSIX or development checkout.

Start a local q process

q -p 5000

The driver sends editor text as q, not ANSI SQL. Use qSQL and normal q expressions such as select from trade where sym=`AAPL.

Example SQLTools connection

{
  "name": "local kdb",
  "driver": "KDB",
  "server": "localhost",
  "port": 5000,
  "username": "",
  "password": "",
  "database": ".",
  "connectionTimeout": 30
}

database maps to a q namespace for explorer queries. Use . for root or values such as .analytics. The command kdb+: Copy Example Global Connection Settings copies a User-settings fragment for persistent SQLTools connections.

Driver features

SQLTools integration over q IPC

Connection and execution

TCP q IPC with optional username/password authentication. Queries are sent synchronously and exactly as written to the remote q process.

Explorer metadata

Object explorer groups for tables, root q views, functions, and columns. Column metadata preserves q meta type, foreign-key, and attribute fields.

SQLTools helpers

Table preview and count support use q sublist for limit/offset previews. Definitions and insert snippets use q-friendly forms.

Result shapes

Tables, keyed tables, dictionaries, vectors, lists, and scalars render in result grids. Nested q values are displayed compactly, and unsafe-width q longs are shown as exact decimal strings.

Default viewer

kdb results panel

Run commands and keybindings

The kdb panel is the default target for kdb+: Run q Script and kdb+: Run Selection. Selection runs send the selected text exactly; with no selection, only the current physical line is sent.

Ctrl/Cmd + Enter
Run selection in kdb panel, replacing an existing result tab.
Ctrl/Cmd + Shift + Enter
Run selection in a new kdb result tab.
Ctrl/Cmd + Alt + Enter
Run the whole q script, replacing an existing result tab.

kdb+: Open kdb Keyboard Shortcuts opens VS Code's Keyboard Shortcuts UI. SQLTools target commands remain available separately.

Replace/new tabs and focus

Default panel runs open a new result tab unless kdb-sqltools.results.kdbPanel.defaultRunMode is set to replace. Explicit Replace and New commands are also available for selections and whole scripts.

Running kdb-panel commands keeps focus in the q editor. Replacing results reveals the existing kdb result tab in its current editor group. New kdb result tabs open beside existing kdb result tabs when one is already present; first-tab placement uses the configured initial view column.

Large-result behavior

Columnar slices, virtual scroll, capped transfers

Columnar panel path

Panel results use a direct columnar path that avoids SQLTools row-object materialization. The q IPC receive path buffers chunks incrementally to reduce copying for large responses.

Visible cell windows

The extension transfers only the visible row and column slice to the webview. The panel virtualizes rows and columns instead of rendering every table cell into the DOM.

Search and sort

Toolbar search runs in the extension against visible columns and returns capped row-match metadata. Sort mode sorts visible columns in the extension and warns before very large row counts.

Honest memory boundary

The q IPC response is still fully materialized in extension memory before display. Use q-side limits for truly massive query results.

Result table tools

Select, copy, export, hide, resize

Selection

Select individual ranges, whole rows, whole columns, or all cells. With no selection, copy/export actions use all cells. Right-click Copy in the table viewport uses the same selected-range path as Ctrl+C/Cmd+C.

Columns and headers

Hide columns for the current panel session, select all, deselect all, and reset hidden columns. Header mode defaults to Drag for session-only column reordering. Switch to Select for whole-column selection or Sort for ascending, descending, and original-order cycles.

Widths and density

The top-toolbar Auto-fit checkbox sizes visible columns from headers and currently rendered cells while scrolling. Manual drag widths and Reset column widths are session scoped. Compact, standard, and comfortable densities persist their own dimensions.

Copy/export formats

Copy formats are CSV, TSV, JSON, NDJSON, HTML, and Markdown. Export formats are CSV, XLSX, TSV, JSON, NDJSON, HTML, and Markdown. XLSX writes a real workbook, rejects Excel sheet-limit overflow, and is export-only.

Warnings and limits

Guardrails without hidden query changes

  • Very large panel results show a non-blocking warning unless suppressed.
  • Large copy/export actions ask for confirmation before materializing output.
  • Large visible-column sorts ask for confirmation unless suppressed.
  • XLSX export enforces Excel's row and column sheet limits.
  • The driver does not add hidden q limits; constrain huge data at the q side.
  • Successful normal result messages stay hidden because the summary already shows row count and elapsed time; errors remain visible.

Limitations

What this extension does not hide

  • No TLS support is implemented by this driver, so no TLS option is exposed.
  • No SQL-to-q translation. Write q and qSQL directly.
  • No fake Parquet export. Parquet is not advertised as available.
  • The kdb panel reduces webview transfer and DOM work, but it does not stream q execution.
  • The q IPC response is fully materialized in extension memory before display.
  • SQLTools' current-query command uses SQL-style statement parsing; select complex q expressions and run kdb+: Run Selection.
  • kdb namespaces are mapped through SQLTools database/schema concepts; they are not SQL catalogs.

Roadmap

Future ideas, not shipped today

Opt-in local data server

A future version may expose current result data through a local-only HTTP server for pandas-friendly CSV and JSON workflows. This is not shipped in 0.2.8.

Interactive plotting

Future charting may compare lightweight plotting options, use downsampling or resampling for large data, and combine multiple query results when they share a common x or time column. This is also not shipped in 0.2.8.
